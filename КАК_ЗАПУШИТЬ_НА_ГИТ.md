# Как запушить проект на GitHub

Ошибка **"Try uploading fewer than 100 files"** появляется при загрузке через **веб-интерфейс** GitHub (кнопка "upload"). Так пушить не нужно — используйте **Git из терминала**: тогда лимита на количество файлов нет.

---

## Вариант 1: Уже есть папка с проектом (рекомендуется)

### 1. Создайте репозиторий на GitHub

1. Зайдите на [github.com](https://github.com) → **New repository**.
2. Название, например: `Web_And_Rasrab` или `east-leaf`.
3. **Не** добавляйте README, .gitignore, лицензию — репозиторий должен быть **пустым**.
4. Нажмите **Create repository**.

### 2. В терминале откройте папку проекта

```bash
cd c:\Users\Bublik\Desktop\Web_And_Rasrab-main
```

### 3. Инициализируйте Git (если ещё не делали)

```bash
git init
```

### 4. Добавьте все файлы и сделайте первый коммит

```bash
git add .
git commit -m "Первый коммит: сайт Восточный Лист на Django"
```

Файлы из `.gitignore` (venv, __pycache__, db.sqlite3, media) в репозиторий не попадут.

### 5. Подключите удалённый репозиторий и запушьте

Замените `ВАШ_ЛОГИН` и `НАЗВАНИЕ_РЕПО` на свои (например: `Bublik` и `Web_And_Rasrab`):

```bash
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_РЕПО.git
git push -u origin main
```

Введите логин и пароль GitHub (или токен, если включена 2FA).

---

## Вариант 2: Репозиторий уже создан и вы в нём что-то делали

Если `git init` уже делали и есть коммиты:

```bash
cd c:\Users\Bublik\Desktop\Web_And_Rasrab-main
git remote add origin https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_РЕПО.git
git branch -M main
git push -u origin main
```

Если `origin` уже есть, но указывает на другой адрес:

```bash
git remote set-url origin https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_РЕПО.git
git push -u origin main
```

---

## Важно

- **Не загружайте проект через кнопку "Upload" на GitHub** — для больших проектов всегда используйте `git add` + `git commit` + `git push` из терминала.
- Благодаря `.gitignore` в репозиторий **не** попадут: `venv/`, `__pycache__/`, `db.sqlite3`, `media/` — так и должно быть.

---

## Если просят логин/пароль при push

- Логин — ваш GitHub username.
- Пароль — **не** пароль от аккаунта, а **Personal Access Token**:
  1. GitHub → Settings → Developer settings → Personal access tokens.
  2. Generate new token, отметьте `repo`.
  3. Скопируйте токен и вставьте вместо пароля при `git push`.
